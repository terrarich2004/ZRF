# Zilda Resource Format (ZRF) — Руководство по созданию контента

ZRF (Zilda Resource Format) — это наш собственный компактный и типизированный язык разметки для описания игровых блоков, предметов и рецептов в движке Zilda. Он пришел на смену устаревшему и громоздкому формату `.bid`. 

ZRF избавляет от дублирования кода, поддерживает строгие типы данных (больше никаких кавычек для Enum-значений вроде `Opaque` или `Pickaxe`) и позволяет описывать несколько сущностей в одном файле.

---

## 1. Структура ресурс-пака

Каждый пак ресурсов Zilda лежит в папке `assets/packs/<ИмяПака>/`. Типичная структура выглядит так:

```text
assets/packs/MyAwesomePack/
├── pack.json             # Манифест пака (задает namespace)
├── blocks/               # Файлы блоков (*.zrf)
│   ├── stone.zrf
│   └── wood.zrf
├── items/                # Файлы предметов (*.zrf)
│   └── tools.zrf
└── textures/             # Текстуры пака
    ├── blocks/
    └── items/
```

### Манифест `pack.json`
Файл определяет пространство имен (`namespace`). Все ID ваших блоков и предметов в ZRF должны начинаться с этого пространства имен.
```json
{
  "name": "MyAwesomePack",
  "namespace": "my_pack"
}
```
*Если ваш namespace — `my_pack`, то ID блоков должны быть вида `my_pack:stone`, `my_pack:boards` и т.д. Движок не загрузит блок с чужим namespace.*

## 2. Основы синтаксиса ZRF

Формат спроектирован так, чтобы код писался легко и читался с первого взгляда.

- **Комментарии**:
  - `// Однострочный комментарий`
  - `/* Многострочный комментарий */`
- **Объявление сущности**:
  `def <Тип> <ID> { ... }`
  Где `<Тип>` — это `Block` или `Item`.
- **Компоненты (Секции)**:
  Начинаются строго с символа `@` и содержат набор полей в фигурных скобках. Пример: `@Props { ... }`.
- **Поля**:
  Записываются в формате `имя_поля: значение`. Перенос строки служит разделителем (запятые или точки с запятой в конце строк ставить **не нужно**).
- **Типы данных**:
  - Строки: Обязательно в двойных кавычках (`name: "Super block"`).
  - Числа: Целые или с плавающей точкой (`hardness: 1.5`, `count: 4`).
  - Булевые: `true` или `false`.
  - Перечисления (Enums): Пишутся **без кавычек** (`render: Opaque`, `tool: Pickaxe`).

## 3. Описание Блоков (`def Block`)

Каждый блок объявляется через `def Block <ID>`. Внутри блока вы настраиваете его поведение, физику, текстуры и рецепты с помощью компонентов.

### Пример описания сложного блока (светящаяся руда с крафтом):

```typescript
def Block my_pack:ruby_ore {
    name: "Ruby Ore"

    // Базовые физические свойства
    @Props {
        render: Opaque           // Тип рендера: Opaque, Cutout, Transparent, Water
        solid: true              // Физический ли это блок
        collision: true          // Будет ли игрок сталкиваться (по умолчанию равно solid)
        stack_size: 64           // Максимальный размер стака в инвентаре
    }

    // Текстуры для разных граней блока
    @Textures {
        all: "textures/blocks/ruby_ore.png" // Быстрый способ задать одну текстуру на все стороны
        
        // Либо можно настроить каждую грань отдельно:
        // top: "textures/blocks/ruby_ore_top.png"
        // bottom: "textures/blocks/ruby_ore_bottom.png"
        // side: "textures/blocks/ruby_ore_side.png" // для всех боковых граней
        // north, south, east, west // для конкретных компасных направлений
    }

    // Параметры прочности и добычи
    @Break {
        hardness: 3.0            // Прочность (земля = 0.5, обсидиан = 50.0)
        tool: Pickaxe            // Инструмент для добычи: Pickaxe, Axe, Shovel, Hoe, Sword
        tier: Iron               // Минимальный тир инструмента: Wood, Stone, Iron, Diamond
    }

    // Выпадающие предметы при добыче блока
    @Drop {
        render: Sprite           // Вид дропа в мире: Cube (блок), CrossedQuads (растения), Sprite (2D)
        survival_item: "my_pack:ruby_crystal" // Какой предмет выпадет
        survival_chance: 1.0     // Шанс выпадения (1.0 = 100%)
    }

    // Излучение света (актуально для лавы, факелов, светящегося камня)
    @Light {
        emit: 12                 // Уровень светимости (от 0 до 15)
        color: "255,20,20"       // Цвет освещения (в формате "R,G,B" от 0 до 255 или Hex "#ff1414")
    }

    // Логика разрушения/опадания (например, для листвы деревьев)
    @Decay {
        requires_neighbor: "my_pack:wood" // Блок, рядом с которым этот блок не будет разрушаться
        max_distance: 4                  // Максимальное расстояние до соседа
    }
}
```

## 4. Описание Предметов и Инструментов (`def Item`)

Предметы в игре могут быть обычными материалами, едой или полноценными инструментами с прочностью и уроном.

### Пример обычного предмета (ресурс):
```typescript
def Item my_pack:ruby_crystal {
    name: "Ruby Crystal"

    @Item {
        texture: "textures/items/ruby.png"
        category: Material       // Категория: Material, Food, Tool, Block, Ingredient
        stack_size: 64
        model: Sprite            // Модель рендера в руке: Sprite (плоский) или Voxel (объемный)
    }
}
```

### Пример инструмента (кирка):
```typescript
def Item my_pack:ruby_pickaxe {
    name: "Ruby Pickaxe"

    @Item {
        texture: "textures/items/ruby_pickaxe.png"
        category: Tool
        stack_size: 1
        model: Voxel             // Инструменты красиво смотрятся объемными
    }

    @Tool {
        kind: Pickaxe            // Тип инструмента: Pickaxe, Axe, Shovel, Hoe, Sword
        tier: Diamond            // Прочность/Тир материала: Wood, Stone, Iron, Diamond
        speed: 8.0               // Множитель скорости копания блоков
        damage: 5.0              // Урон по мобам
        durability: 1561         // Запас прочности инструмента
    }
}
```

## 5. Описание рецептов Крафта (`@Craft`)

Компонент `@Craft` можно добавить внутрь любого `def Block` или `def Item`. Рецепты бывают двух типов: **Shapeless** (бесформенный) и **Grid** (сетка 3x3).

### Shapeless рецепт (ингредиенты можно класть в любое место верстака)
```typescript
    @Craft {
        type: Shapeless
        ingredients: "my_pack:ruby_crystal, my_pack:ruby_crystal" // Ингредиенты через запятую
        count: 1                 // Сколько предметов скрафтится на выходе
    }
```

### Grid рецепт (требует строгой раскладки в сетке 3x3)
```typescript
    @Craft {
        type: Grid
        count: 1
        // Рисунок рецепта: 3 строки по 3 символа, разделенные запятыми
        // Пробел означает пустую ячейку
        pattern: "RRR, S , S " 
        
        // Маппинг символов на ID предметов/блоков
        keys: "R:my_pack:ruby_crystal,S:my_pack:stick"
    }
```

## 6. Справочник компонентов (директив) и параметров

Компоненты (начинающиеся с `@`) служат декларативными директивами для сборки свойств. Ниже приведен полный список всех доступных параметров для каждого компонента.

### Компонент `@Props` (Свойства блока)
*Используется только в `def Block`.*
- `render` (обязательное) — тип рендера блока. Допустимые значения: `Opaque`, `Cutout`, `Transparent`, `Water`.
- `solid` (обязательное) — является ли блок физически твердым (проходит ли сквозь него игрок). Значения: `true`, `false`.
- `collision` (необязательное) — просчитывать ли столкновения. По умолчанию совпадает со значением `solid`.
- `creative_visible` (необязательное) — показывать ли блок в меню креативного режима. Значения: `true` (по умолчанию), `false`.
- `stack_size` (необязательное) — максимальный размер стака в инвентаре. По умолчанию `64`.
- `place_as` (необязательное) — ID другого блока (строка в кавычках), который будет фактически поставлен в мир при клике.
- `interact` (необязательное) — тип интерактивности. Например, `"crafting"` для открытия окна верстака.

### Компонент `@Textures` (Текстуры блока)
*Используется только в `def Block`.*
- `all` — задает одну текстуру на все 6 граней блока.
- `side` — задает текстуру на боковые грани (север, юг, восток, запад).
- `top` — текстура верхней грани.
- `bottom` — текстура нижней грани.
- `north`, `south`, `east`, `west` — текстуры для конкретных боковых сторон.
- `particle` — текстура разрушения (частиц) блока.

### Компонент `@Colors` (Цветовые маски блока)
*Используется только в `def Block`.*
Позволяет накладывать цветовой оттенок поверх текстуры. Принимает цвет в формате RGB (`"255,255,255"`) или Hex (`"#ffffff"`).
- Параметры те же, что и у `@Textures`: `all`, `side`, `top`, `bottom`, `north`, `south`, `east`, `west`.

### Компонент `@Icon` (Иконка в интерфейсе)
*Используется только в `def Block`.*
Настраивает внешний вид блока в инвентаре, если он отличается от стандартной 3D-модели.
- `texture` (строка) — путь к текстуре иконки.
- `color` (строка) — цвет заливки (Hex или RGB).

### Компонент `@Break` (Параметры добычи)
*Используется только в `def Block`.*
- `hardness` (число) — время разрушения блока. По умолчанию `0.6`.
- `tool` (Enum) — рекомендуемый инструмент. Значения: `Pickaxe`, `Axe`, `Shovel`, `Hoe`, `Sword`.
- `tier` (Enum) — минимально необходимый тир инструмента. Значения: `Wood`, `Stone`, `Iron`, `Diamond`.

### Компонент `@Light` (Излучение света)
*Используется только в `def Block`.*
- `emit` (число) — сила света от `0` до `15`. По умолчанию `0`.
- `color` (строка) — цвет излучаемого света (RGB или Hex). По умолчанию `"255,255,255"`.
- `animation` (строка) — имя анимации мерцания света.

### Компонент `@Drop` (Выпадение при разрушении)
*Используется только в `def Block`.*
- `render` (Enum) — режим отображения выброшенного предмета в мире. Значения: `Cube` (по умолчанию), `CrossedQuads`, `Sprite`.
- `survival_item` (строка) — ID предмета, который выпадет.
- `survival_chance` (число) — шанс выпадения от `0.0` до `1.0`. По умолчанию `1.0`.

### Компонент `@Decay` (Разрушение без поддержки)
*Используется только в `def Block`.*
- `requires_neighbor` (строка) — ID блока, удерживающего данный блок от распада.
- `max_distance` (число) — максимальный радиус поиска опорного блока. По умолчанию `4`.

### Компонент `@Item` (Свойства предмета)
*Используется только в `def Item`.*
- `texture` (обязательное, строка) — путь к текстуре предмета.
- `category` (Enum) — вкладка/категория. Значения: `Food`, `Tool`, `Material`, `Block`, `Ingredient` (по умолчанию `Material`).
- `stack_size` (число) — размер стака. По умолчанию `64`.
- `model` (Enum) — вид рендера в руке игрока. Значения: `Sprite` (плоский) или `Voxel` (объемный). По умолчанию `Voxel` для инструментов, `Sprite` для всего остального.
- `heal` (число) — количество восстанавливаемых сердец (только для категории `Food`).

### Компонент `@Tool` (Свойства инструмента)
*Используется только в `def Item`.*
- `kind` (Enum) — тип инструмента. Значения: `Pickaxe`, `Axe`, `Shovel`, `Hoe`, `Sword`.
- `tier` (Enum) — тир материала. Значения: `Wood`, `Stone`, `Iron`, `Diamond`.
- `speed` (число) — коэффициент скорости добычи. По умолчанию `2.0`.
- `damage` (число) — урон в ближнем бою. По умолчанию `1.0`.
- `durability` (число) — запас прочности. По умолчанию `59`.

### Компонент `@Craft` (Рецепт крафта)
*Можно использовать как в `def Block`, так и в `def Item`.*
- `type` (Enum, обязательное) — тип рецепта. Значения: `Shapeless`, `Grid`.
- `count` (число) — количество получаемых предметов. По умолчанию `1`.
- `ingredients` (строка, для Shapeless) — список ингредиентов через запятую.
- `pattern` (строка, для Grid) — 3 строки рисунка раскладки через запятую.
- `keys` (строка, для Grid) — биндинг символов к ID ресурсов.

## 7. Что делать, если игра крашится (Система логов)

Мы внедрили Fabric-style систему отчетов об ошибках. Больше никаких долгих поисков забытой скобки в логах терминала. 

Если в `.zrf` файле допущена ошибка:
1. Игра мгновенно остановится.
2. В папке `logs/` запишется подробный отчет `crash-ГГГГ-ММ-ДД_ЧЧ-ММ-СС.log`.
3. Игра **сама откроет этот файл** в вашем Блокноте или текстовом редакторе по умолчанию.

### Пример лога при забытом символе `@`:
Если вы напишете `Craft {` вместо `@Craft {`, вы увидите такой лог:

```text
=================================================================
                  ZILDA ENGINE CRASH REPORT                      
=================================================================

Time: 2026-06-09_20-56-34
Resource File: C:\Users\markn\Desktop\Zilda\assets\packs\ZildaPack\blocks\boards.zrf
Error: did you forget '@' before component 'Craft' at line 17, column 5?

Code Fragment:
-----------------------------------------------------------------
  17 |     Craft {
     |     ^
-----------------------------------------------------------------

Backtrace:
did you forget '@' before component 'Craft' at line 17, column 5?
```

Стрелочка `^` указывает на точное место ошибки, а текст прямо подскажет, что вы пропустили знак `@`.
